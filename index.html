Arrays & Prefix Sums

Topic 01 • Accumulators & Offsets
1 Solved

Prefix sums are an essential tool for O(1) query-based subarray computations. By storing cumulative values, we can answer range sum queries in constant time.

🔍 Recognition Pattern

Use this when asked for sum or differences of subarrays, especially if you need to calculate left-side vs right-side cumulative totals repeatedly.

⚠️ Common Pitfalls

  • Index out of bounds on margins (use sizes of N+1 or handle index 0 explicitly).
  • Neglecting off-by-one calculations on left/right boundaries.
# Prefix Sum Computation Template
def compute_prefix_sums(nums):
    n = len(nums)
    prefix = [0] * (n + 1)
    for i in range(n):
        prefix[i + 1] = prefix[i] + nums[i]
    return prefix

Two Pointers & Sorting

Topic 02 • Sliding & Converging Bounds
1 Solved

Sorting the array simplifies many search problems, enabling two converging pointers to sweep from opposite directions to locate elements satisfying constraints (e.g. sums) in O(N) time.

🔍 Recognition Pattern

Look for terms like "triplet sum", "pair sum", "find three numbers", or search problems in sorted inputs where elements can be skipped to avoid duplicate comparisons.

⚠️ Common Pitfalls

  • Forgetting to sort the array first.
  • Including duplicate elements in the results (always increment pointers past duplicate values).
# Converging Two-Pointer Template
nums.sort()
left, right = 0, len(nums) - 1
while left < right:
    curr_sum = nums[left] + nums[right]
    if curr_sum == target:
        # Match found!
        left += 1
    elif curr_sum < target:
        left += 1
    else:
        right -= 1
