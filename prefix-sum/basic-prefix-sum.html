Basic Prefix Sum

Jun 27, 2026 · 10 min read · Beginner

Prefix sums are one of those ideas that seem almost too simple to be useful - until you realize half the "hard" array problems on LeetCode boil down to them. The core insight is dead simple: pre-compute cumulative sums once, then answer any range query in constant time. Once this pattern clicks, you'll start recognizing it everywhere.

What is a Prefix Sum?

A prefix sum array is a derived array where each element at index i stores the sum of all elements from the original array from index 0 up to i. Think of it as a running total - at each position, you know the cumulative sum of everything before it.

The mental model

Imagine you're tracking your bank balance over time. Each day's balance already includes all previous deposits and withdrawals. If you want to know how much you spent between Tuesday and Friday, you don't go back and add up each day - you just subtract Tuesday's balance from Friday's. That's a prefix sum in action.

Given an array nums = [2, 4, 1, 3, 5], the prefix sum array is [0, 2, 6, 7, 10, 15]. Notice it's one element longer - that leading 0 is intentional and saves you from a bunch of edge-case headaches.

Building the Prefix Array

The construction is straightforward. You create a new array of size n + 1 (where n is the length of the input), initialize the first element to 0, and fill in each subsequent position by adding the current element to the previous prefix sum.

Python 
def build_prefix_sum(nums):
    n = len(nums)
    prefix = [0] * (n + 1)

    for i in range(n):
        prefix[i + 1] = prefix[i] + nums[i]

    return prefix

# Example
nums = [2, 4, 1, 3, 5]
print(build_prefix_sum(nums))
# Output: [0, 2, 6, 7, 10, 15]

Warning

The most common bug with prefix sums is an off-by-one error. If your prefix array has size n + 1 with prefix[0] = 0, then the sum of subarray nums[i..j] is prefix[j + 1] - prefix[i], not prefix[j] - prefix[i]. Draw it on paper the first few times - it's worth the thirty seconds.

Range Sum Queries in O(1)

This is where the prefix sum earns its keep. Once you've built the prefix array, answering "what's the sum of elements from index i to index j?" becomes a single subtraction.

Python 
def range_sum(prefix, i, j):
    """Sum of nums[i..j] inclusive, using 0-indexed prefix array."""
    return prefix[j + 1] - prefix[i]

# Using our earlier prefix: [0, 2, 6, 7, 10, 15]
# Sum of nums[1..3] = prefix[4] - prefix[1] = 10 - 2 = 8
# Which is 4 + 1 + 3 = 8 ✓

prefix = [0, 2, 6, 7, 10, 15]
print(range_sum(prefix, 1, 3))  # 8

When to reach for this pattern

  • When you need to compute subarray sums or perform range queries repeatedly.
  • When you need to search for subsets or subarrays that equal a specific target.
  • When array values are static (no updates), but queries are extremely frequent.
